## 概念

《皇帝的信息流》把安徒生童话 **《皇帝的新衣》** 改写成一个当代平台控制界面。项目关注的不是单纯的道德选择，而是一个媒介问题：

> 如果大家其实已经感觉到某件事是假的，是什么样的系统让这个谎言仍然保持公开稳定？

玩家进入宫廷媒体办公室，成为 **宫廷发布编辑**。在皇帝游行开始前，玩家只有六次发布行动，用来决定哪些官方声明、证据、评论、私人记录和直接声音可以进入公共记录。游戏把熟悉的童话转化为一个关于信息可见性、官方措辞、公众犹豫、AI 改写和集体确认的可玩系统。

## 摘要

在安徒生的故事里，谎言能够维持，并不是因为布料真的可信，而是因为公开怀疑是危险的。官员称赞自己看不见的东西，人群表演一致，而孩子那句直白的话之所以重要，是因为它打破了公共发言的规则。《皇帝的信息流》把这个结构重新媒介化为一个中英双语 Next.js 网页游戏：一个每条发布都会带来后果的宫廷发布台。

最终项目包含标题页、新手教程、操作台、游客与账号游玩、云存档、档案、成就、音频、多结局、本地确定性规则，以及带离线 fallback 的宫廷 AI 回应。在线 AI 可以丰富改写、评论、对话、指引和报告，但核心玩法和结局仍由本地规则计算，保证可复现和可测试。

## 玩法

玩家不是简单地选择“说真话”或“不说真话”。玩家要决定真话如何经过一个受控媒体系统。

| 步骤 | 玩家行动 | 叙事功能 |
|---|---|---|
| 1 | 选择来源 | 裁缝、大臣、公众评论和孩子声音分别揭示故事的不同部分。 |
| 2 | 预览后果 | 玩家看到一条发布可能如何改变证据、安全、群众怀疑或宫廷注意。 |
| 3 | 发布或接受改写 | 直接证据可能被宫廷 AI 软化；更安全的话术也可能保护编辑。 |
| 4 | 阅读信息流 | 评论、AI 指引、指标变化和公共记录展示故事正在如何变化。 |
| 5 | 抵达结局 | 最终结果取决于什么被看见、什么被传播，以及谁还保有发布权。 |

每局都很短，但不是线性：六次行动、更多可选路线，以及多个代表不同媒介状态的结局。

## 系统设计

### 信息来源

发布台把童话中的人物和社会力量转化为编辑来源。

| 来源 | 系统中的作用 |
|---|---|
| 裁缝室 | 官方声明、布料话术、织布机证据，以及欺骗的第一层。 |
| 大臣报告 | 公共权威、私人恐惧，以及宫廷合法性内部的矛盾。 |
| 公众评论 | 群体重复、孤立怀疑、共同确认和评论可见性。 |
| 孩子的声音 | 最简单的真话，也最危险，因为它直接且难以被宫廷话术吸收。 |

### 指标

游戏使用六个直观指标，让玩家不需要读规则手册也能理解每次选择为什么重要。

| 指标 | 含义 |
|---|---|
| 证据 | 可验证材料进入公共记录的程度。 |
| 宫廷压力 | 官方权威让公开反对变得困难的程度。 |
| 传播 | 一条发布扩散的范围，不论它是否真实。 |
| 群众怀疑 | 人们是否能看见别人也在犹豫。 |
| 你的安全 | 编辑暂时免于宫廷报复的程度。 |
| 宫廷警戒 | 宫廷距离收回发布台控制权还有多近。 |

### 结局

结局不是简单的好坏分支，而是不同的信息状态：

- 赞美保持公开、怀疑停留私下时，谎言可以继续稳定。
- 证据即使存在，也可能被更安全、更容易重复的话术压低。
- 真话传播过快时，编辑可能在公共记录稳固前先被暴露。
- 最强的结果需要证据、共同怀疑和孩子的直白声音保持连接。

## 宫廷 AI

AI 是故事世界的一部分。它以 **宫廷叙事引擎** 的形式出现，是一个冷静、制度化、以稳定为优先的系统。它可以：

- 建议更安全的发布行动；
- 把直接内容改写成更含糊的话术；
- 生成公众评论和信息流反应；
- 打开突发交流；
- 在结局后生成报告。

AI 层并不是规则引擎。数值变化、行动锁定、成就和结局都由本地规则计算。如果没有配置 API key，应用会返回确定性 fallback 文案，因此项目在课堂演示、提交和归档场景中仍然完整可玩。

## 媒介框架

项目把制作实践与课程概念连接起来：

| 概念 | 游戏中的体现 |
|---|---|
| 媒介融合 | 文字、界面、规则、评论、音频、AI 文本、账号和档案成为同一个可玩作品。 |
| 重新媒介化 | 十九世纪童话被转化为社交信息流 dashboard 和发布后台。 |
| 超文本与电子文学 | 玩家通过来源、发布、记录和结局构建路径，而不是阅读固定顺序。 |
| 参与式文化 | 公众评论可以形成共同确认，但参与仍然受平台可见性影响。 |
| 再混合文化 | 《皇帝的新衣》与审核、排序、AI 改写和平台治理重新组合。 |

## 技术架构

项目是一个使用 React 19 和 TypeScript 编写的 Next.js 16 应用。代码把叙事表层和确定性规则引擎分开，使游戏在 AI 文本变化时仍然可测试、可复现。

| 层级 | 主要文件 |
|---|---|
| 页面和路由 | `src/app/` |
| 主发布台循环 | `src/app/dashboard/dashboard-client.tsx` |
| 行动数据和来源区域 | `src/lib/game-data.ts` |
| 规则引擎和结局逻辑 | `src/lib/game-rules.ts` |
| 突发交流 | `src/lib/dialogue.ts` |
| 双语文案和术语 | `src/lib/i18n.ts` |
| AI-compatible 客户端和 fallback | `src/lib/ai.ts` |
| 账号和云存档 | `src/lib/auth.ts`, `src/lib/profile.ts` |
| 测试 | `src/**/*.test.ts`, `e2e/*.spec.ts` |

## 功能

- 英文与简体中文双语界面。
- 六次行动的叙事循环，包含预览、确认和最终报告。
- 多结局、成就、档案记忆和复玩目标。
- 宫廷 AI 建议、改写、生成评论、对话和报告。
- 在线 AI 不可用时的确定性 fallback 文案。
- 游客模式使用浏览器存档。
- 可选账号登录和 SQLite 云存档。
- 标题页、发布台、突发交流、档案和结局音频场景。
- Vitest 单元/API 测试和 Playwright 流程/视觉检查。
- Docker 部署支持。

## 建议演示流程

1. 打开标题页，展示中英文切换。
2. 开始新值班，跟随教程进入发布台。
3. 发布一次偏宫廷安全的行动，展示稳定路线。
4. 发布一次推进证据的行动，展示风险、评论和指标变化。
5. 展示宫廷 AI 指引或改写弹窗。
6. 触发或展示突发交流。
7. 进入结局，解释最终报告如何回应路线。
8. 打开档案，展示结局、成就和复玩记忆。

## 本地运行

```bash
pnpm install
pnpm dev --hostname 127.0.0.1 --port 7987
```

打开：

```text
http://127.0.0.1:7987
```

没有 AI key 也可以运行。在线生成不可用时，服务端路由会返回确定性 fallback 文案。

## 环境变量

如果需要在线 AI 或本地账号数据库，复制 `.env.example`：

```bash
cp .env.example .env.local
```

重要变量：

| 变量 | 用途 |
|---|---|
| `OPENAI_API_KEY` | 服务端 API key。不要提交真实 key。 |
| `OPENAI_BASE_URL` | OpenAI-compatible API 地址。 |
| `OPENAI_MODEL` | AI 辅助路由使用的模型。 |
| `OPENAI_PROVIDER_MODE` | `responses` 或 `chat`。 |
| `OPENAI_HTTP_TRANSPORT` | 默认 `fetch`；本机网络兼容问题可使用 `curl`。 |
| `DATA_DIR` | SQLite 数据目录。 |
| `AUTH_COOKIE_SECURE` | 本地 HTTP 使用 `false`，HTTPS 部署使用 `true`。 |

接口诊断：

```bash
OPENAI_API_KEY=... pnpm ai:diagnose
```
